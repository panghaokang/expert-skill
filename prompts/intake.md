# 企业专家信息录入

## 开场白

```
我来帮你蒸馏这位企业专家的知识。只需要回答几个问题，每个都可以跳过。
```

---

## 问题序列

### Q1：专家称呼

```
这位专家怎么称呼？（花名、昵称或英文名都行）

例：qing-yun / zhang-san / Alex
```

- 生成的 slug 统一用 `-` 连接
- 中文自动转拼音（"青云" → `qing-yun`）
- 英文小写（"Alex Wang" → `alex-wang`）

---

### Q2：基本信息

```
用一句话描述他的基本信息——公司、职级、职位、专业领域，想到什么写什么，跳过也行。

例：字节 2-1 后端工程师 微服务架构
```

解析以下字段（缺失的留空）：
- **公司**
- **职级**
- **职位**
- **专业领域**

---

### Q3：专长类型

```
他的专业能力属于哪种类型？

  [A] 诊断专家 (troubleshooter)
      擅长故障诊断与排查，能从现象快速定位根因

  [B] 架构专家 (architect)
      擅长系统设计与技术选型，能从全局视角做架构决策

  [C] 审核专家 (reviewer)
      擅长代码/方案审查，能快速发现缺陷与风险点

  [D] 决策专家 (decision_maker)
      擅长在不确定条件下做权衡与优先级判断

  [E] 运维专家 (operator)
      擅长系统运维与稳定性保障，掌握大量操作经验

选一个最接近的，也可以跳过（默认按诊断专家处理）。
```

---

### Q4：专业深度描述（可选但推荐）

```
用 2-3 句话描述他的专业特点——他最擅长解决什么问题？
他的判断框架是什么？别人遇到什么问题时第一个想到他？

例：他是我司微服务治理的 go-to person。任何服务间通信的诡异超时
问题，他上来先查连接池状态和 GC 停顿，三板斧下来基本都能定位。
核心信条是"先看系统指标再看业务日志"。
```

## 确认汇总

```
信息汇总：

  👤  {称呼}
  🏢  {公司} {职级} {职位}（若未填则省略）
  🎯  专长领域：{领域}（若未填则省略）
  🧩  专长类型：{类型}
  💡  专业描述：{描述}（若未填则省略）

确认无误？（确认 / 修改 [字段名]）
```
