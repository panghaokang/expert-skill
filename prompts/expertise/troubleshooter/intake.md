# Troubleshooter — 诊断专家摄入

## 补充问题

在基础信息录入后，针对诊断专家追问：

### Q：诊断方法论

```
你排查陌生故障时，标准的前三步是什么？有没有那种"每次必查"的固定检查项？

例：
1. 先看有没有最近的变更（部署、配置、流量切换）
2. 再看监控大盘的核心指标（QPS / 延迟 / 错误率）
3. 然后二分法隔离问题范围（网络/应用/存储）
```

### Q：常用工具

```
你日常排查问题最常用哪些工具/命令/平台？

例：kubectl logs --tail、arthas、PromQL、Grafana 某看板、jstack
```

### Q：经典案例

```
说一个你最拿手的故障类型——那种别人搞不定但你一看就知道怎么回事的。
你是怎么查的？根因通常是什么？
```
