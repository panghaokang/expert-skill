# Troubleshooter — 诊断知识分析

## 类型专项提取

在通用 expertise_analyzer.md 的基础上，重点关注：

### 故障模式库

从原材料中提取他遇到并处理过的故障类型：
- 故障类型名称
- 典型症状
- 根因分类
- 出现的频率（经常/偶尔/仅一次）

```
输出格式：
故障模式：
  - 类型：[名称]
    症状：[描述]
    常见根因：[列表]
    频率：[经常/偶尔/仅一次]
```

### 诊断决策链

提取他从现象到结论的完整排查路径：
- 初始症状
- 假设列表（他考虑了哪些可能）
- 验证步骤（他怎么排除或确认每个假设的）
- 最终定位

特别注意他**排除过的假设**——这些往往比最终发现的根因更体现专业知识。

```
输出格式：
诊断路径：
  症状：[描述]
  → 假设1：[描述] → 验证：[怎么排除的]
  → 假设2：[描述] → 验证：[怎么确认的]
  → 最终定位：[根因]
```

### 嗅觉检查 (Smell Tests)

他一上来就查的固定检查项，不管问题描述是什么：
- 最近的变更记录
- 核心监控指标
- 连接池/线程池状态
- GC / 内存状况
- 网络连通性

```
输出格式：
固定检查项（按顺序）：
  1. [检查项] — 为什么先查这个：[理由]
  2. [检查项] — 为什么先查这个：[理由]
```
