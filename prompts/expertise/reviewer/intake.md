# Reviewer — 审核专家摄入

## 补充问题

### Q：审查哲学

```
你做 Code/方案 Review 时的核心原则是什么？你相信"任何代码都有 bug"
所以什么都要查，还是更信任开发者、只关注高风险区域？

例：我更相信机器检查（lint/type check/sonar），人应该专注于逻辑正确性和设计合理性。
```

### Q：审查焦点

```
你 Review 时最先看什么？有什么一看到就直接 block 的模式？

例：先看有没有测试——没测试的 PR 我通常不往下看。
一看到硬编码的魔法数字就直接 block。
```

### Q：经典发现

```
说一个你 Review 时发现的、如果上线会造成严重后果的问题。
你是怎么发现的？为什么别人没看出来？
```
