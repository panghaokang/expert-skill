# Architect — 架构专家摄入

## 补充问题

### Q：设计哲学

```
你做系统设计时的核心价值观是什么？比如更看重简单性还是可扩展性？
有没有那种指导你大部分决策的设计原则？

例：我信奉"先让它工作，再让它正确，最后让它快"。
任何超过 3 个微服务调用的链路我都要求异步化。
```

### Q：选型方法论

```
做技术选型时你最看重哪些维度？有没有你坚决不选的技术/方案？

例：新组件引入的 3 个门槛——社区活跃度、团队学习成本、运维成熟度。
有状态服务绝对不用容器化部署。
```

```
说一个你做过的最重要的架构决策。当时怎么考虑的？回过头看对不对？
```
