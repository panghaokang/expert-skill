# Operator — 运维专家摄入

## 补充问题

### Q：运维哲学

```
你做运维的核心信条是什么？比如你相信"自动化一切"还是
"复杂系统需要人的判断"？你对"稳定"的定义是什么？

例：我信"监控先行"——没有监控的上线等于闭眼开车。
稳定不是不出问题，是出了问题能 5 分钟内发现、10 分钟内回滚。
```

### Q：监控与告警

```
你最关注哪些监控指标？告警阈值怎么定的？有没有那种
"指标看起来正常但你知道要出事了"的经验信号？

例：别看 CPU，看 GC 停顿时间。CPU 高不一定有问题，
GC 频繁停顿一定有问题。数据库连接数突然下降往往是
雪崩的前兆。
```

第一步做什么？什么情况下直接回滚不查原因？
